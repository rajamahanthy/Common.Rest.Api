Architecture Overview

✅ Layered Architecture: Domain → Application → Infrastructure
✅ Shared Cross-Cutting Concerns: Middleware (CorrelationId, ExceptionHandling), Health Checks, Auth, OpenTelemetry
✅ Dual Database Support: SQL Server or InMemory (dev/test)
✅ API Versioning: Built-in with Asp.Versioning
✅ Health Checks: Liveness (/health/live) and Readiness (/health/ready) probes


Cross-Cutting Concerns
Implemented in RestApi.Shared:

✅ Authentication - Microsoft Identity Web with dev fallback
✅ OpenTelemetry - Distributed tracing with Azure Monitor
✅ API Versioning - Asp.Versioning (URL + Header)
✅ Health Checks - Liveness/Readiness probes
✅ Resilience - Polly + HttpResilience policies
✅ Structured Logging - Correlation IDs for request tracking

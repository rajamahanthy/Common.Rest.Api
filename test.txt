Environment Used: The Common.Rest.Api microservices solution is built on .NET 10 and targets a cloud-native architecture with Azure Cosmos DB as the primary data store. Development occurs in Visual Studio Community 2026 on Windows, with local testing supported via the Azure Cosmos DB Emulator. The solution implements a modular layered architecture across three microservices (Address, Hereditament, and SurveyData), each following Domain-Driven Design principles with separate Application, Domain, Infrastructure, and API layers. Integration tests are configured to connect to either a local Cosmos DB emulator or cloud-based instances via connection string configuration, ensuring flexibility across development, testing, and production environments.
Deployment Approach & Technical Demo Snapshot: The application employs a modern containerized deployment strategy leveraging Azure infrastructure, with hosted services that initialize Cosmos DB resources on startup. Each REST API service exposes standardized endpoints secured through authentication middleware, with health check endpoints (/health/live and /health/ready) supporting Kubernetes-style orchestration. The technical architecture demonstrates a production-ready microservices implementation featuring resilience patterns, OpenTelemetry observability, standardized validation, and OpenAPI documentation. For example, the Address service provides RESTful CRUD operations on address documents with partition-key-based queries, while the unified infrastructure layer abstracts Cosmos DB complexity through repository and unit-of-work patterns, allowing seamless repository swaps or migration to alternative data stores without impacting business logic.
